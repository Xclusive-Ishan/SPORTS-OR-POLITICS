Sport vs Politics Document Classifier

Machine Learning Text Classification Project

Introduction

This project explores the design of a simple text document classifier that categorizes text snippets as either Sport or Politics. The goal was to experiment with multiple ML models and compare their performance.

Data Collection

  • Scraped news articles using newspaper3k and BeautifulSoup
  • Collected ~150 sports and ~160 politics articles
  • Manual labeling and cleaning
  • Lowercasing, URL removal, tokenization
  • Stopword removal using NLTK

Final Dataset:

  • Sports: 147
  • Politics: 152

Dataset Analysis

  • Average words per doc: Sports ~220 | Politics ~290
  • Balanced class distribution (~3% difference)
  • Vocabulary overlap observed (e.g., "lead", "party", "win")

Feature Representation

  • Bag of Words (BOW)
  • TF-IDF
  • Unigrams + Bigrams (n-grams)

Machine Learning Models

1. Multinomial Naive Bayes

Fast, simple, classic text classifier.

2. Logistic Regression

Best performing model overall with TF-IDF + bigrams.

3. Linear SVM

Strong performance on sparse high-dimensional data.

Experimental Setup

  • 70% Training / 30% Testing
  • Multiple shuffled runs
  • Metrics: Accuracy, Precision, Recall, F1 Score

Results

Model Accuracy Precision Recall F1 Score
Naive Bayes 0.82 0.83 0.80 0.81
Logistic Regression 0.88 0.89 0.87 0.88
Linear SVM 0.87 0.88 0.86 0.87

Logistic Regression performed best overall. TF-IDF + bigrams consistently improved performance.

Limitations

  • Small dataset size (~300 documents total)
  • Vocabulary overlap between classes
  • No stemming/lemmatization
  • No deep learning models used

Conclusion

Logistic Regression achieved the highest performance, closely followed by SVM. Naive Bayes performed reasonably well given its simplicity. For real-world deployment, significantly more data and advanced techniques would be required.
